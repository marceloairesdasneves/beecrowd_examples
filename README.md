## 📌 About this Repository
This repository contains my solutions for competitive programming and logical puzzles solved on the [beecrowd](https://www.beecrowd.com.br/) platform (formerly URI Online Judge) using **Java**.

Solving these problems is a core part of my routine to sharpen computational thinking, algorithm efficiency (runtime and memory optimization), and clean code practices. Each solution is structured to follow solid programming fundamentals.

## 🛠️ Performance & Best Practices Applied
* **Scanner Optimization:** Using standard `Scanner` or high-performance `BufferedReader` / `StringTokenizer` depending on the input size to avoid *Time Limit Exceeded* (TLE).
* **Float Precision:** Handling `double` and `float` outputs strictly to meet the formatting constraints (e.g., using `Locale.US` and `System.out.printf`).
* **Conditionals Optimization:** Structuring nested logic in selection tests (like 1035 and 1037) to stop early and consume fewer CPU cycles.
